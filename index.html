Перелік основних команд якi потрібно записати:

  • git config --global user.name blabla - Команда для вводу імені користувача
  • git config --global user.email blabal@bla- команда для вводу емейла користувача
  • git status-Перераховує всі нові або змінені файли, які потребують фіксації
  • git add-Додаємо файли в коміт. Зазвичай використовується «.» або «*» для додавання усіх файлів. Щоб додати один файл – вказуємо його ім'я та розширення.
  • git commit-а допомогою цієї команди ми виконуємо «коміт» змін. Тобто, вказуємо, що поточний зліпок проекту потрібно зберегти.
  • git log-Перераховує комміти, зроблені в репозиторії у зворотному до хронологічного порядку, — останні коміти знаходяться вгорі.
  • git checkout-Дозволяє переміщатися між гілками, створеними командою git branch.
  • git branch-Це свого роду "менеджер гілок". Вона вміє перераховувати ваші гілки, створювати нові, видаляти та перейменовувати їх.
  • git push-Пушим» або відправляємо в репозиторій наші зміпки проекту
  • git pull-Команда для отримання змін з репозиторію.
  • git merge-Команда git merge використовується для злиття однієї або кількох гілок у поточну. Потім вона встановлює покажчик поточної гілки на результуючий коміт
  • git clone-Це утиліта командного рядка Git для вибору існуючого репозиторію та створення його клону.

    • gitignore файл

  • gitignore файл -це простий файл ,який потрібений для приховування файлів та папок від системи контролю версій Git.

    • Advanced

  • git revert-Відкочує коміт шляхом створення нового, зміни в якому обернені до того комміту, який потрібно скасувати. Таким чином, відкат буде видно в історії як черговий коміт.
  • git reset-Скасує всі коміти після заданого, залишаючи всі зміни у робочій директорії.
  • git rebase-Rebase стискає всі зміни в один "патч". Потім він інтегрує патч у цільову гілку.
  • git cherry-pick-Команда git cherry-pick використовується для перенесення окремих коммітів з одного місця репозиторію до іншого, зазвичай між гілками розробки та обслуговування.
  • git stash-Команда git stash призначена для того, щоб помістити поточні зміни, які ви виконали у файлах, в окреме сховище та повернути файли до вихідного стану
